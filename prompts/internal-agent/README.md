# Internal-agent prompt — the Telegram bot's identity

This folder owns the source-of-truth prompt for the bot's own personality and behaviour. It is distinct from `./CLAUDE.md` at the repo root (which talks to the *developer* agent that builds MVPClaw).

## The four `CLAUDE.md` files in the system

Mixing them up is the single most common mistake — keep them distinct.

| # | Path | Owner | Committed | Edited by the runtime agent |
|---|---|---|---|---|
| 1 | `./CLAUDE.md` (repo root) | maintainer | yes | no |
| 2 | `./prompts/internal-agent/CLAUDE.md` (this folder) | maintainer | yes (source only) | no |
| 2′ | `~/.mvpclaw/workspaces/default/CLAUDE.md` (runtime copy of #2) | maintainer (sync overwrites) | no | no |
| 3 | `~/.mvpclaw/workspaces/default/CLAUDE.local.md` | runtime agent (via `memory_append`) | no | yes (append-only) |
| 4 | `./CLAUDE.local.md` (gitignored) | human developer | no | never |

- **#1** is "rules for the AI building MVPClaw" — frozen stack, code quality, CLI-first principle, source-as-docs.
- **#2** is "rules for the bot itself" — the file in this folder. The maintainer edits the source here; the orchestrator syncs it to **#2′** at boot.
- **#3** is the bot's own scratchpad — append-only, secret-redacted, size-capped + rotated.
- **#4** is the human dev's local Claude Code preferences — never the bot's business.

The Claude CLI provider runs with `cwd = ~/.mvpclaw/workspaces/default/`, NOT the repo root. That's how Claude Code finds **#2′** and **#3** and not **#1**.

## The L0–L9 prompt composition hierarchy

The composer (lands in ticket P15) assembles every model call in this exact order. Static-first, dynamic-last — the only order that hits Anthropic's prompt cache.

| Layer | Path | Owner | Loaded by composer | Writable by |
|---|---|---|---|---|
| **L0** Static preamble | hardcoded in `src/prompts/preamble.ts` | maintainer | every call, section 2a | source only |
| **L1** Project memory (dev) | `./CLAUDE.md` (repo root) | maintainer | Claude Code dev only — NOT loaded by composer | maintainer |
| **L2** Project memory (runtime) | `./prompts/internal-agent/CLAUDE.md` → synced to runtime workspace | maintainer | every call, section 2d | maintainer (sync overwrites) |
| **L3** Agent runtime memory | `~/.mvpclaw/workspaces/default/CLAUDE.local.md` | agent (via `memory_append`) | every call, section 2e | agent (append-only) + user (full edit via CLI) |
| **L4** Per-chat memory | SQLite `chat_memory` table | agent + user | every call, section 2f (only the active chat's row) | agent (`memory_append --scope chat`) + user |
| **L5** Skill description | `./skills/<name>/SKILL.md` YAML frontmatter | maintainer | every call, section 2b — metadata only | maintainer |
| **L6** Skill body | `./skills/<name>/SKILL.md` body | maintainer | section 2b only when `/skill-name` forces it | maintainer |
| **L7** Skill references | `./skills/<name>/references/*.md` | maintainer | NOT preloaded — agent reads on demand via tool | maintainer |
| **L8** Tool descriptions | generated by `ToolRegistry.describe()` | tool authors | every call, section 2c | tool authors |
| **L9** Human dev local | `./CLAUDE.local.md` (gitignored) | developer | Claude Code dev only — NOT loaded by composer | developer |

### Composition order (verbatim from ARCHITECTURE §12.1)

```text
1.  Tools block                                  (alphabetical by tool name)
2.  System messages, in order:
    2a. Static preamble                          (L0)
    2b. Skill fragments                          (L5 always; L6 only when forced)
    2c. Tool descriptions appendix               (L8)
    2d. Project memory (runtime CLAUDE.md)       (L2)
    2e. Agent runtime memory (CLAUDE.local.md)   (L3)
    2f. Per-chat memory                          (L4)
3.  Conversation history                          (sliding window from P14)
4.  Current user turn                             (the inbound message)
```

Anthropic cache breakpoints land at BP1 (end of tools block), BP2 (end of 2c), BP3 (end of 2f), and BP4 (optional, end of 2nd-to-last assistant turn). Static-first means the prefix up through BP3 is byte-stable across turns within a chat, which is what produces cache hits.

## Sync contract (boot-time)

Implemented in ticket P16. Pseudo-code:

```text
on every boot:
  if mtime(./prompts/internal-agent/CLAUDE.md) > mtime(~/.mvpclaw/workspaces/default/CLAUDE.md):
    copy ./prompts/internal-agent/CLAUDE.md → ~/.mvpclaw/workspaces/default/CLAUDE.md

  NEVER touch ~/.mvpclaw/workspaces/default/CLAUDE.local.md during sync — it's
  a different file owned by the runtime agent.
```

## How to edit this prompt safely

1. Edit `./prompts/internal-agent/CLAUDE.md` in the repo.
2. `pnpm check` — it changes nothing for the test suite, but confirms the project is still healthy.
3. Restart the bot (`pnpm dev`). Boot-time sync overwrites the runtime copy with your edit.
4. Test the change end-to-end with `mvpclaw send --channel cli-inject --chat-id <test-id> --text "…"` and read the trace to confirm the prompt body changed.
